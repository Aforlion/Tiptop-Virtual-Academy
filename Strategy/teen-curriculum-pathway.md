## Executive Summary
This educational blueprint defines the curriculum sequencing, learning outcomes, and assessment rubrics for expanding Tiptop Virtual Academy to support teen learners (Ages 13-16). We establish a transition from basic blocks to industry-aligned systems engineering and code logic.

---

## Program Objective
To build a workforce-readiness pathway that equips teens with real-world programming competencies, software engineering principles, and a portfolio of verifiable GitHub projects.

---

## Learner Analysis
- **Learner Profile:** Teens aged 13-16 with varying coding backgrounds (from beginners to advanced hobbyists).
- **Existing Knowledge:** Basic understanding of loops, variables, and logic (often learned through Scratch or senior gaming brackets).
- **Learning Barriers:** High distraction, low engagement with dry technical theory, and frustration with overly childish learning environments.
- **Learning Goals:** Build custom web apps, publish code, and earn certificates for school or career portfolios.

---

## Competency Framework

### 1. Knowledge (What they must know)
- State machines and application routing.
- Database relational design and double-entry auditing systems.
- Git, GitHub collaboration, and version control.

### 2. Skills (What they must do)
- Write modular, clean code using modern frameworks (Next.js/React).
- Create safe database schema migrations (SQL constraints, triggers, RLS).
- Deploy active web applications to cloud servers.

### 3. Behaviors (How they must perform)
- Conduct asynchronous code reviews and participate in discussion threads.
- Practice independent problem-solving using technical documentation.

---

## Curriculum Structure

### Module 1: Web Foundations & Git (Weeks 1-4)
- Sequence: Terminal commands, basic HTML/CSS layouts, and local Git repository setup.
- Practical: Host a personal landing page on GitHub Pages.

### Module 2: Relational Databases & Backend Logic (Weeks 5-8)
- Sequence: SQL basics, RLS policies, custom stored procedures, and triggers.
- Practical: Design a simulated bank transaction ledger in SQL.

### Module 3: Frontend Integration & Next.js (Weeks 9-12)
- Sequence: React hooks, Next.js page routers, client state management, and API calls.
- Practical: Integrate the bank ledger database with a modern dashboard UI.

---

## Certification Pathway
- **Certification Title:** Junior Web Architect (Level I)
- **Eligibility:** Completion of Modules 1-3.
- **Assessment Requirements:** Score >= 80% on the final assessment and submit a verified GitHub repository link of a capstone project.
- **Certification Criteria:** Checked and verified by an Admin / Teacher during live portfolio reviews.

---

## Assessment Strategy
- **Continuous:** Module-specific coding labs and automated pull request feedback.
- **Summative:** A 60-minute comprehensive exam (MCQ, system design questions) combined with a live, 10-minute project presentation in the Live Classroom Dock.

---

## Industry Alignment
The curriculum matches requirements for entry-level Junior Developer positions, focusing on React, SQL, and Git. Verifiable portfolios provide proof of capabilities, bypassing traditional academic prerequisites.

---

## Risks
- **Drop-off during database modules:** SQL theory can feel abstract. Recommendation: Keep assignments project-focused (e.g. building game databases instead of abstract rows).
- **Varying skill levels:** Some teens will finish early. Recommendation: Include stretch tasks (e.g. implementing OAuth).

---

## Recommendations
- **Rec. 1 (High Confidence):** Replace traditional worksheets with interactive coding labs that run unit tests on student submissions.
- **Rec. 2 (Medium Confidence):** Partner with tech startups to offer mock interview sessions for advanced students.
