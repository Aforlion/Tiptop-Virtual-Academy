## 1. Objective
To define the operational standards, curriculum milestones, competency guidelines, and verification rules required to award the **Junior Web Architect (Level I)** certification to students in the Teen bracket (Ages 13–16).

---

## 2. Competency Framework
Candidates must prove proficiency across three core areas before a certificate can be generated:

### A. Knowledge
* Relational database design and security standards (including PostgreSQL RLS policies).
* Client-server communication models (Next.js server-side vs. client-side renders).
* Asynchronous state synchronization models.

### B. Technical Skills
* Writing clean, modular React components using Tailwind CSS / vanilla styles.
* Creating database schema tables, keys, triggers, and migrations.
* Executing version control routines (Git commits, branch merges, pull requests).

### C. Collaborative Behaviors
* Contributing constructive feedback in academy channel discussion threads.
* Presenting capstone code projects during live cohort sessions.

---

## 3. Curriculum Requirements & Milestones
Students must complete three sequential learning modules:

* **Module 1: Web Foundations & Git**
  * Practical Deliverable: A personal static webpage hosted on GitHub Pages.
* **Module 2: Relational Databases & Backend Logic**
  * Practical Deliverable: A simulated double-entry transaction database ledger designed in SQL.
* **Module 3: Frontend Integration & Next.js**
  * Practical Deliverable: Integration of the database ledger with a responsive frontend dashboard.

---

## 4. Certification & Audit Criteria
The system will permit certification generation only when the following criteria are met:

1. **Continuous Assessment:** Completion of all module assignments with a combined score >= 80%.
2. **Capstone Submission:** A verified, public GitHub repository link submitted via the student dashboard.
3. **Admin Verification:** The portfolio must be reviewed and approved by an Administrator or Teacher.
